## What's parked on `wip/stats-persistence`

**Engine surface (typecheck-clean, no tests yet):**

- `packages/engine/src/types/stats.ts` ‚Äî `PlayerGameStats` (15 fields:
  pass/rush/recv/defense), `PlayerSeasonStats` (extends with `gamesPlayed`),
  `emptyPlayerGameStats(playerId)` factory.
- `packages/engine/src/games/stats.ts` ‚Äî `deriveGamePlayerStats(game, league)`.
  Pure & deterministic (no PRNG). Distributes team `GameResult` stats by
  tier+role using largest-remainder integer splits:
  - QBs: 93/7 split for QB1/QB2 of pass volume (attempts, completions,
    yards, TDs, INTs)
  - RBs/FB: 70/30 (or 62/25/13) of carries + rushing yards/TDs
  - Receivers (WR/TE/RB): Pareto-ish target distribution `[0.28, 0.20,
    0.16, 0.13, 0.10, 0.08, 0.05]`, ranked by `receivingWeight √ó keySkillAvg`
  - Defense: 55 tackles/team distributed LB/DB/DL = 55/30/15%; sacks split
    EDGE 60% / interior 40%; INTs ‚âà 60% of opp turnovers, DBs 80% / LBs 20%
  - Stat conventions: `team.turnovers` = offensive give-up; `team.sacks` =
    defensive credit (matches NFL box score reading)
- `packages/engine/src/season/stats.ts` ‚Äî `seasonStatsForLeague(league)`
  walks regular-season + playoff games and aggregates. Plus
  `playerSeasonStats(league, playerId)` lookup.
- Exports threaded through `season/index.ts` and engine public API
  (`@gmsim/engine`).

## Things deliberately deferred

- **No PRNG / random noise in distribution.** Same game + same rosters
  produces identical stat lines. This is fine for placeholder; play-by-play
  resolution is a much later (out-of-scope) system.
- **No "did the player play?" gating.** Injured players can still
  receive stat lines. The injury propagation slice put injuries in
  `Player.injury` but the game sim doesn't yet skip injured players.
- **No special teams stats.** K/P/LS produce no stat lines.
- **No OL stats** (sacks allowed, pancakes). The OL position group is
  just absent from output.
- **Per-game stat lines aren't stored** ‚Äî every read re-derives via
  `seasonStatsForLeague`. That's a deliberate North-Star call (see
  `types/game.ts` comment about not bloating `GameResult`). Inspector
  must `useMemo`.

## Conventions

- Conventional Commits (CONTRIBUTING.md)
- Keep a Changelog format
- Sync versions across all packages: `pnpm version:sync <new>`
- Determinism-first: every random call goes through the seeded PRNG.
  Stats derivation has no PRNG so determinism is automatic.
- North Star: hidden ratings never displayed numerically to player.
  The inspector is a dev tool exception. Stats *are* player-visible
  in real life, so this surface is North-Star-compatible.
